概述 Overview

點點印｜作品管理與創作流程重塑

#B2C #RWD #Redesign

用戶 管理作品的核心區域，依照不同創作階段（如編輯中、已完稿、已出版...等）分類顯示作品列表。使用者可透過主動操作（如篩選作品類型、查看作品狀態）快速定位目標作品，並根據當前狀態執行最適行動（如繼續編輯、加入購物車、作品欣賞等）。

Team | TinTint 點點印

Role | UIUX Designer

Collaborators | UIUX Designer(1)、PM(1)、FE(1)、BE(1)、CS(2)

[調整背景]
為統一整體網站的設計風格，並簡化舊版中不必要的頁面和資訊，先針對 整體資訊架構進行盤點與梳理，移除重複或多餘內容， 聚焦呈現使用者真正需要的資訊。


在整理過程中發現「作品列表」的操作流程不夠直覺，使用者常需多層點擊才能完成任務。經與團隊夥伴討論後，決定重新設計該區塊，同時 全面對齊設計系統 2.0 的樣式與規範，提升整體體驗一致性與操作效率。

我的作品列表

主要操作 CTA 點擊率提升

將「繼續編輯」等 關鍵行動按鈕直接置於作品卡片上，使用者能更快識別並執行操作，有效降低思考與點擊成本，進而提升 CTA 點擊率達 45%。

簡化步驟和資訊，操作效率提升

當附加功能收合後，用戶無需跳轉至其他頁面即可完成操作。根據 用戶回饋與行為數據顯示，操作步驟減少後，使用流程更為直覺，轉換率提升了 15%，用戶平均完成時間也縮短了 30%。

客服相關問題回報量下降

簡化介面後，用戶更容易理解作品狀態與後續操作，客服關於「作品去哪了」、「如何進入編輯器」的 詢問明顯減少。

痛點與觀察 Pain Point & Task

資訊層級混亂，重點不易辨識

  • 資訊列表的文字缺乏層級差異，所有資訊呈現方式相似，難以快速抓到關鍵。
  • 顯示過多「次要資訊（如圖片數量、檔案大小）」，反而分散注意力，使用者需額外花時間篩選重要資訊。

操作路徑繁瑣，效率低落

  • 缺乏明確的「主要動作（如「繼續編輯」）」指引，使用者需進入詳細頁才能操作，流程冗長。
  • 操作選項如「複製」、「刪除」 需進入下一層畫面才能執行，對有大量作品的使用者來說操作成本高。

介面佈局擁擠，視覺負擔大

  • 作品卡片圖片比例過大且垂直排列導致資訊密集，空間使用不佳。
  • 卡片切割區塊過多、缺乏留白與視覺節奏，使使用者在瀏覽時感到壓迫與混亂。

行動指引不清，主次不分明

  • 「繼續編輯」、「作品欣賞」等行動按鈕樣式不夠明顯 ，易與周圍資訊混淆。
  • 行動區缺乏層次感與視覺突出效果，重要操作按鈕位置偏下且需捲動，易被忽略。

設計優化重點 Solution

雖無大規模調研，仍透過數據、客服回饋與競品分析，歸納出操作痛點與共通模式。據此重塑資訊架構與動線，聚焦提升操作效率與資訊清晰度，優化整體體驗。

▍資訊層級清晰

明確標示作品狀態與主要操作，減少使用者思考負擔。

▍操作動線精簡

將核心按鈕前置至卡片，避免多層操作，提升效率。

▍視覺與排版一致

優化對比與留白、統一文字對齊與間距，讓畫面更整潔易讀。

▍介面更易上手

移除不必要資訊與干擾元素，降低學習門檻。

資訊架構 Define

為了讓產品在擴充功能或優化介面時具備一致且清晰的結構，先對盤點現有功能，釐清現行設計中可能存在的混亂或規格不一致問題。接著將功能邏輯整理為資訊架構與規格文件，幫助團隊更快理解整體產品脈絡與使用者操作邏輯。

User Flow Ideate

讓流程問題具象化的第一步

我先繪製了 UI Flow，將頁面路徑與層級視覺化，協助團隊辨識舊版的痛點，例如操作層級過深、「繼續編輯」需多次點擊才能到達對應頁面，以及按鈕主次層級不明確。藉由這些洞察，我們優化了介面結構與資訊架構，使層級更清晰、流程更直覺。

Wireframe

建立設計共識，優化用戶認知成本

藉由低擬真原型與團隊快速溝通配置邏輯，驗證操作假說。本次優化核心在於「資訊去蕪存菁」，將非必要干擾收納至次層級，確保介面簡潔且重點清晰。

▍整合附加功能，減少視覺噪音

將次要的操作按鈕（如：詳細資訊、複製、刪除等）收納至作品卡片的 「More」選單。透過功能收疊，保持列表介面清爽，讓使用者專注於核心任務。

▍過濾冗餘資訊，聚焦核心需求

移除對一般使用者非必要的後台數據，僅保留關鍵資訊。若需查看細節，則引導至 「詳細資訊 Modal」 呈現，有效降低首屏的認知負擔與視覺壓力。

UI Design

統一新版設計風格

  • 風格更貼近品牌個性
    新版介面採用柔和配色與簡約排版，傳遞溫暖、小清新的品牌氣質，營造親和、舒適的使用氛圍。

  • 分層概念
    使用了卡片式設計，將內容明確區分（如左側側邊欄、作品列表區），讓使用者直覺理解資訊架構。

  • 簡化視覺
    採用柔和配色與低對比，減少過度裝飾，依內容需求排版，營造舒適且聚焦的閱讀體驗。

  • 一致性與突出層次
    標題、選單、圖像區塊明確，資訊層次感提升讓介面更直覺易懂。

主次分明，強調核心資訊

[Before] 資訊密集且缺乏重點

使用者難以一眼辨識關鍵內容，需耗費額外時間搜尋。

[After] 結構化重塑，直覺操作
  • 狀態化管理： 透過 Tab 區分「編輯中、已完成」，並依狀態（如：截稿日）彈性調整顯示色彩與資訊，強化視覺提醒。
  • 資訊層級優化： 運用不同字重區分標題與輔助資訊（如：作品編號），讓畫面重點聚焦於縮圖與標題。
  • 簡化操作負擔： 將複製、刪除等次要功能收納至 More 選單，減少視覺噪音。

按鈕優先級明確，行動導向清楚

[Before]
缺乏明確操作按鈕，使用者需點擊作品卡片並進入下一層頁面才能執行操作，容易忽略或增加操作時間。

[After]
每張作品卡片 根據狀態不同，皆設有明顯的主要行動按鈕，如編輯中的作品是「繼續編輯」，以完稿的作品是「加入購物車」可直接操作，無需跳轉頁面，提升使用效率。

具體做法
  • 主要 CTA 使用高對比色，與背景形成明顯區隔。
  • 簡化畫面資訊，讓用戶更聚焦在不同作品卡片上。

More 選單設計，互動方式更直覺

[Before]
用戶需點擊作品卡片才可以看到動作按鈕，且所有選項都直接顯示在頁面上，增加視覺負擔且干擾理解主要操作。 [After]
More 按鈕（三個點圖示）讓額外功能像是複製、刪除...等，不佔用主要空間，但仍然容易存取。

具體做法
  • 次要功能收納在 More 按鈕中，保持畫面簡潔。
  • More 選單使用簡潔的圖標與文字，確保易讀性與可操作性。

空間感優化，讓畫面更易讀

▍視覺節奏與留白優化
  • 核心洞察：原版資訊過於擁擠，缺乏適當留白，導致使用者閱讀壓力較大且難以掃描。
  • 優化方案：透過調整區塊間距（如不同產品類型的分隔）與統一排版節奏，即使在品項眾多時，畫面仍能維持清晰結構，有效降低視覺疲勞並提升資訊掃描效率。明確標示作品狀態與主要操作，減少使用者思考負擔。
▍操作效能與批次流暢度
  • 核心洞察： 原有的大量操作邏輯不明確，影響使用者管理作品的效率。
  • 優化方案： 重新規劃「大量選取」按鈕的權重與位置，提供更直覺的批次操作入口。透過一致的介面邏輯，提升加入購物車與作品整理的整體效率。

創建作品 流程優化

模組化入口，建立流暢起點

針對舊版頁面跳轉產生的體驗破碎、側邊導覽干擾視線等痛點進行重塑。透過介面減法，確保用戶在單一場景內能毫無壓力地啟動創作任務。

  • 情境連續性： 改以 Modal 取代傳統跨頁跳轉，讓用戶在不脫離原產品場景的前提下啟動任務，大幅降低操作時的迷失感。
  • 淨化視覺噪音： 移除冗餘的側邊導覽與破碎的輸入框，引導用戶將注意力集中在「當下設置」，避免因過多無關選項導致決策疲勞。
  • 降低進入門檻： 透過簡化操作路徑與明確的視覺引導，緩解新手進入編輯器前的焦慮感，確保體驗流程的穩定與流暢。

縮短物理與心理路徑

將關鍵決策前置並整合於單一場景，顯著降低了用戶從「瀏覽產品」到「動手做」的轉換成本。

理解效率轉化

透過視覺語彙的統一，降低用戶的學習負擔，將視覺的一致性實質轉化為「操作速度」。

低風險的探索體驗

非強制性的彈窗設計，讓用戶能隨時關閉並回到原頁面，減少「擔心出錯」的心理阻力，提升嘗試動機。

「映畫本」作為首波實驗點

本次優化採「試點計畫」形式，優先於映畫本導入速編流程，旨在驗證「決策前置」對完稿動機的影響。

  • 追蹤映畫本在改版後的轉化率與停留時間，並與舊有流程之產品進行橫向對比。
  • 「先選模板、後進編輯」的引導式邏輯，有效驗證了能緩解用戶面對空白畫布的焦慮，讓完稿目標變得更具體、更可達成。
設計反思

觀察用戶在鎖定主題後，對「彈性換版型」的接受度，作為未來優化編輯器自由度的基礎。

若映畫本之完稿率有顯著提升，將計畫將此「引導式設計」與「Modal 創建入口」全面推行至全站產品線。

引導式設計，降低創作壓力

針對新手對於「如何開始排版」的畏難心理，將主題選擇由編輯器內移至流程前段。透過「視覺先行」的策略，讓用戶在正式進入編輯前即建立清晰的成品預期。

▍決策前置與具象化

將主題範本選擇移至進入編輯器前，提供強大的視覺錨點。用戶能立即在腦中勾勒出成品樣貌，將不確定的「摸索」轉化為明確的「執行」。

▍消除空白焦慮

避免用戶進入編輯器後面對空白畫布的無助感。透過預設的主題風格，賦予用戶創作信心，顯著提升從「開啟」到「動手做」的轉換率。

▍動態排版彈性

在鎖定主題風格的前提下，仍保留核心的版型切換功能。用戶可根據照片構圖彈性調整每頁佈局，在快速完稿與個性化創作間取得最佳平衡。

Style Guide

視覺層次清晰，資訊更有條理

Before

元素密集且顏色偏重，缺乏層級引導，使用者視線需來回瀏覽，增加負擔。

After

提升卡片與背景的對比度，並運用醒目色引導使用者聚焦操作行為，使畫面閱讀更輕鬆直覺。

色彩與對比更簡潔，符合現代設計趨勢

Before

桃紅與深灰的強烈對比讓視覺較為尖銳，區塊分隔不明確，卡片切割過細，降低整體閱讀性。

After

採用柔和、低飽和的配色（細緻灰＋蜜桃橘），營造溫潤親和的視覺氛圍。同時強化區塊結構，更符合現代設計。

字體設計與排版更精緻，提升可讀性

Before

文字與按鈕區塊略顯擁擠，字級與間距偏小，降低閱讀舒適度。

After

透過更清晰的標題層級、適當的字距與留白，提升整體排版節奏；同時將關聯內容歸納在同一視覺區塊，強化資訊結構與視線引導。

強化介面質感與親和力

Before

版面風格偏早期網頁設計，邊角銳利、分隔生硬，缺乏視覺層次與柔和感。

After

透過圓角應用於按鈕、卡片與選單，營造更友善且現代化的風格；搭配適度留白與陰影，提升界面呼吸感與整體質感，使用者更易聚焦於主要內容。

UI Kit

Takeaway

▍使用者其實不需要那麼多資訊

原以為越多資訊越清楚，實際上過多細節反而造成干擾。精簡後，使用者反而更快找到重點。

▍將關鍵動作放在「對的位置」，減少用戶多想

把「繼續編輯」、「加入購物車」、「作品欣賞」...等關鍵動作按鈕直接放在卡片上，讓動作成為視覺焦點，提升使用者對下一步的確定感。

▍資訊結構重整帶來清爽感

優化後的畫面更整齊、分明，不少使用者回饋「比較好找」、「不會眼花」，證明視覺層次的影響力很大。

▍選擇不該是壓力，而是指引

畫面上不需要一次呈現所有可能，反而要在適當時機給出明確建議。讓使用者在不同狀態都知道「現在該做什麼」，附加功能做收合，減少認知負擔。
