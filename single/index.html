客服編輯相關問題回報量下降 35%

過去客服收到許多用戶詢問「不知道怎麼完稿下單」、「找不到已上傳照片」等問題，在新流程導入後一個月內，相關回報數量大幅下降，顯示導引與流程簡化有效降低學習成本。

用戶更願意完成作品

新流程上線一個月後，後台數據顯示從進入編輯器到完成作品下單的轉換率顯著提升，棄單率下降 25%。推測與操作路徑清晰化、步驟導引有關，降低了使用初期的流失。

行動裝置使用者完成率成長

優化後的編輯器特別針對響應式體驗進行設計調整，提升了手機與平板上的操作順暢度。根據 GA 數據，行動裝置用戶的作品完成率從 18% 提升至 32%，使用佔比也略微提升。

9 成用戶在回饋中提到操作更直覺

透過 CS 整理的回饋和實際找幾位用戶訪談，發現超過 9成的用戶提到「容易使用」、「流程清楚」、「畫面變得俐落不笨重」等關鍵字，反映新版本成功改善編輯器的使用引導問題。

痛點與觀察 Pain Point & Task

舊版的問題

🔺操作效率受限於螢幕與滾動
在行動裝置上，因螢幕較小且介面需頻繁上下滑動，使用者難以快速切換功能，增加點擊次數與操作負擔。

🔺關聯操作分散各處
像「新增」、「選物件」、「加雕刻」在上方，照片池與新增按鈕卻藏在下方，導致操作流程斷裂、學習成本高。

🔺預覽體驗不穩定
滾動切換功能時，作品位置容易偏移和抖動，干擾細節調整與即時預覽。

🔺資訊層級不清
樣式、顏色、雕刻等選項未明確區分，容易讓使用者搞不清楚目前編輯狀態。

設計優化重點 Solution

為解決上述問題，我重新規劃操作流程與介面，讓體驗更清晰流暢。

  • 統一操作區塊位置
    將照片池固定於左側，並整合編輯工具列，減少視線與操作的跳轉。
  • 優化照片新增流程
    使用者可直接於編輯區上傳與替換照片，流程更簡潔。
  • 重整按鈕與工具排列邏輯
    依使用場景分類，提升操作直覺性。
  • 提升視覺一致性與精緻度
    使用低飽和色系、加強層次感（圓角、陰影等），打造更現代的介面風格。
  • 調整字體與元件間距
    改善閱讀與點擊體驗，提升整體易用性。

資訊架構 Define

先盤點現有功能，統整規格並解決不一致之處，建立規格文件與資訊架構。同時讓團隊成員快速理解產品結構，加速開發決策。

User Flow Ideate

精簡流程並重新梳理編輯體驗

資訊結構與佈局定案

將「儲存、進階編輯」等低頻功能收納至 Header，並根據功能重要性分配權重。透過視覺減法釋放核心編輯空間，將 User Flow 轉化為直覺的作業區塊，確保使用者能專注於核心創作任務。

變換產品流程優化

數據顯示用戶多在相似產品間切換，舊版採「全品項展示」且「直接跳轉」的設計，常導致用戶疑惑與流程中斷。
優化後改採意圖導引式 Modal，透過分段提示強化切換前的心理預期，確保過程透明並完全由用戶主導，大幅提升操作流暢度。

從操作邏輯到結構佈局

透過 Wireframe 確立架構功能

  • 結構定案：將 User Flow 轉化為具體的「三欄式佈局」（照片池、編輯區、外觀樣式），極大化核心作業空間，直覺化操作動線。
  • 功能簡化與評估：重新審視功能必要性，將低頻率操作（如儲存、進階編輯）收納至 Header 導覽列，降低視覺噪音並減少認知負擔。
  • 層級優先權：在排除視覺干擾的狀態下，透過佈局定義資訊的讀取順序，確保使用者能專注於當前的編輯任務，並維持跨產品線的操作一致性。

UI Design

作品編輯區

強化響應式：固定高度範圍

  • 明確定義不同段點區間裝置的畫面呈現。
  • 優化後的設計更加直覺、層次分明，並且加強了視覺焦點與操作便利性，使得使用者在編輯過程中能更專注於作品本身，帶來更流暢的體驗。
-照片池

三欄式高效佈局：重塑編輯動線

▍針對快製產品建立直覺的作業架構
  • 結構減法：將照片池移至左側側欄，解決舊版頻繁捲動導致的操作斷層，極大化中央預覽視野。
  • 任務聚焦：右側整合樣式控制台，確保所有屬性調整皆能在「一屏內」完成，無需切換分頁。
  • 產品區隔：針對卡片、海報等輕量化產品，提供短路徑、高效率的編輯體驗，與進階版（Editor2）功能明確分流。
-導覽列

聚焦編輯任務，簡化導覽列選項

▍舊版：分散專注力
延用官網全站式導覽（Header）與頁尾（Footer），過多的行銷資訊與無關連結導致使用者在編輯過程中容易分心。

▍新版：聚焦核心任務
針對編輯器任務重新梳理導航架構，僅保留編輯、預覽與購買等核心功能。同時導入防誤觸機制，當使用者欲跳轉回官網時，透過離開提示 Modal 進行二次確認，有效防止未存檔資料遺失。

-變換產品

聚焦同系列變換產品，降低誤用風險

原本提供完整產品列表供自由切換，但觀察到多數使用者僅在同系列間變換，較少進行跨品類操作（如木框畫換成相片筆記本），且「變換產品」屬於附加、非高頻功能。

因此新版僅保留並收合「同系列產品」選項，讓介面更聚焦、降低誤用，同時保留照片套用至相似產品的彈性。

-Tooltips 教學

增加友善的教學即時引導，降低學習門檻

針對特定產品的部分功能（如 海報 與 木框畫 的「新增雕刻」），我們加入情境式 Tooltips，在使用者初次進入編輯器適時提示，協助理解操作方式，降低學習門檻。

此機制保留彈性，未來新增功能也能沿用，不影響版面配置，提升功能可見性與使用率。

-功能列

介面工具列優化

▍強化頁面功能 Bar 體驗
  • 將功能列移至編輯區上方，操作更直覺。
  • 優化體驗，減少滑動與視線移動。
  • 功能分區明確，有助快速完成編輯。
  • 點擊對應的元間區塊會顯示對應的Bar。
▍樣式與操作一致化
  • 統一文字與照片工具列樣式，符合新版設計系統。
  • 文字 Modal 支援字體、字級、顏色與對齊設定。
  • 無需回外層調整，Modal 內即可完整編輯。
▍功能命名與行為邏輯統一
  • 維持一致命名與開啟邏輯，降低使用者認知負擔。將原有不同名稱的設定按鈕（如「編輯作品」、「基本內容」…）統一為「作品資訊」。
  • 作品資訊 Modal 中統一可編輯項目為：作品名稱與作品說明。

Style Guide

視覺層次清晰，資訊更有條理

Before

介面元素較密集，顏色較重，資訊層次不夠明顯，且視線需要來回移動，增加認知負擔。

After

採用更明確的分區設計

  • 分區優化：縮減照片管理區，極大化編輯區留白與內容層級。
  • 視覺減法：統一工具欄與選單佈局，降低視覺雜訊。
  • 高效編輯：依產品自動對應樣式選項，加速選取流程。

色彩與對比更簡潔，符合現代設計趨勢

Before

採用深灰與橘色的深色背景，並透過漸層營造立體感，但強烈的色彩對比容易造成視覺疲勞，同時功能區塊之間的區隔不夠明顯。

After

  • 改用更柔和、現代的配色（淺米色＋細緻灰＋低飽和橘色），整體視覺更溫暖親和。
  • 背景顏色變淺，使內容與圖片更突出，避免背景喧賓奪主。
  • 畫面更柔和卻仍保持清晰可讀性。

圓角與留白強化層次，細節更具質感

Before

版面呈現早期網頁設計風格，以銳利邊角與硬性分隔為主，缺乏柔和的視覺層次。

After

  • 元件與區塊改用圓角設計，介面更親和現代。
  • 適度留白並突顯主要內容。
  • 搭配細緻陰影與分層，清楚區隔照片編輯區與工具欄。

字級與排版更精緻，提升可讀性

Before

按鈕與文字區塊稍顯擁擠，字體大小與間距略小，導致可讀性較差。

After

  • 標題更明顯，字距更適中，將關聯性較高的內容集中在同區塊。
  • 按鈕與功能名稱間距拉開，減少擁擠感，提高可讀性。

Takeaway

在專案中，我學會先釐清資訊結構讓操作更順，再透過問對問題找到真正的痛點，同時用清楚的設計語言讓不同部門都能理解並推進。

清晰資訊架構，讓操作更順暢

在優化介面時，我更深刻體會到資訊架構與視覺層級對使用行為的影響。透過整理功能區塊、運用標題與配色引導，我讓畫面更清晰有邏輯，幫助使用者更快理解與操作，提高整體流暢度與編輯意願。

聚焦問題，解決用戶痛點

在用戶訪談中，我透過開放式提問挖掘操作困難的根本原因。例如，有使用者表示「放大縮小會讓人想放棄」，實際問題卻在於功能分散、需頻繁切換。
這讓我體會到，好的設計應從使用者流程與心理出發，透過重整資訊結構降低認知負擔。

溝通設計思維，提升協作效率

透過清晰的設計邏輯與流程圖，高效協助 PM 和工程團隊掌握重點，大幅降低溝通成本與反覆修改。過程中，雖有觀點差異，但我們維持理性溝通並理解彼此立場，這成為跨部門協作中的重要學習。
