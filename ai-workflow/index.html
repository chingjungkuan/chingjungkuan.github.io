AI × Design

知識管理 ｜ 競品分析 ｜ 快速原型 ｜ 埋點規格

為什麼把 AI 整進設計流程

加速、系統化、解放重複性工作

設計師日常有大量重複性高、但判斷門檻低的工作——整理競品資訊、撰寫規格草稿、歸納使用者回饋。將這些任務交給 AI 協作處理，讓我能把注意力放在需要設計直覺與決策的部分。

我的 AI 協作原則

工具是手段，設計判斷才是核心

AI 產出的是草稿與素材，最終的架構決策、視覺判斷與使用者同理心仍由設計師主導。我把 AI 視為一個很快的執行夥伴，而不是替代思考的工具。

知識庫管理系統

個人 PARA 知識庫 × Claude Agent

工具｜Claude Code、Obsidian、GitHub

架構｜PARA（Projects / Areas / Resources / Archive）

用 Claude Agent 建立一套自動化知識管理流程：每日靈感自動爬取並分類、設計靈感整理進 Obsidian 知識庫、重要筆記同步備份至 GitHub。透過客製化 Skill 指令，可直接呼叫 AI 搜尋、新增、歸檔筆記。

Key Outcomes
  • 自動化靈感收集 | 每日定時爬取設計靈感，自動歸入知識庫。
  • 快速筆記存取 | 用自然語言指令搜尋、新增、整理筆記。
  • 跨裝置同步 | 透過 GitHub 備份，隨時在任何裝置取用。

競品 Agent × 功能矩陣自動產出

工具｜Claude Agent、Skill（competitor-analysis）

輸入競品名稱，AI 自動產出功能清單、UX 模式整理與對比矩陣。原本需要 2–3 小時的競品整理，縮短至 30 分鐘內完成初稿，設計師只需專注在判斷與補充。

Key Outcomes
  • 結構化輸出 | 功能清單、UX 模式、對比矩陣一次產出。
  • 大幅縮短前期研究時間 | 初稿從 2–3 小時縮至 30 分鐘。
  • 客製化 Skill | 可針對不同產品類型調整分析框架。

快速原型協作

AI 協作產出互動原型與設計稿

工具｜Claude Code、Figma

在概念發想階段，與 AI 快速迭代 HTML/CSS 原型，驗證佈局與互動邏輯；在正式設計前先用低成本的方式確認方向，減少在 Figma 上反覆修改的時間。

Key Outcomes
  • 概念快速驗證 | 在 Figma 前先確認佈局與互動方向。
  • 降低修改成本 | 早期用 code 原型測試，減少後期翻案。
  • 設計與開發對話更順 | 有可跑的 prototype 更容易溝通規格。

數據埋點規格設計

截圖素材補充中

Analytics Spec Skill × 追蹤規格書產出

工具｜Claude Agent、GA4、Skill（generate-analytics-spec）

設計師通常不熟悉埋點規格的撰寫格式，透過客製化的 Analytics Spec Skill，輸入頁面互動需求，AI 自動產出事件名稱、參數定義與 QA 驗收清單，降低設計與工程的溝通門檻。

Key Outcomes
  • 規格標準化 | 每次輸出格式一致，減少溝通誤差。
  • 設計師也能主導埋點 | 不需等工程師定義，設計端即可產出草稿。
  • 與 GA4 直接對應 | 輸出的事件定義可直接對應 GA4 自訂維度。
