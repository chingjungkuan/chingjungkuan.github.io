痛點與觀察 Pain Point & Task

舊版編輯器的問題

  • 文字與按鈕間距不足 → 降低可讀性，點擊容易出錯。
  • 操作不直覺 → 主要按鈕不明顯，使用者需要額外摸索，學習成本高。
  • 視覺風格過時 → 缺乏現代 UI 元素，層次感不足，長時間使用容易造成視覺疲勞。
  • 資訊架構混亂 → 功能分散，使用者必須頻繁移動視線尋找，操作效率低落。
  • 照片上傳流程不順暢 → 點擊「新增照片」需跳轉頁面後再返回當前編輯畫面，且照片池以左右切換呈現，違反常見的下滑操作習慣，增加理解與操作成本。

我們發現使用者在操作過程中常遇到三大問題：難以快速找到所需資訊、學習成本高、操作流程不直覺。為了釐清根源，我們結合 Personas、Empathy Map 與 User Story Map，具體描繪出不同角色在編輯過程中的需求與行為痛點，並透過用戶訪談與實際操作觀察驗證假設，例如「照片上傳流程過於繁瑣」與「功能按鈕分散導致視線頻繁切換」。

設計優化重點 Solution

為了解決上述問題，我重新規劃了操作流程與介面結構，讓整體體驗更直覺清晰：

  • 統一操作區塊位置：將照片池固定於左側，並整合編輯工具列，減少視線與操作的跳轉
  • 優化照片新增流程：使用者可直接於編輯區上傳與替換照片，流程更簡潔
  • 重整按鈕與工具排列邏輯：依使用場景分類，提升操作直覺性
  • 提升視覺一致性與精緻度：使用低飽和色系、加強層次感（圓角、陰影等），打造更現代的介面風格
  • 調整字體與元件間距：改善閱讀與點擊體驗，提升整體易用性

這些設計有效降低了使用者的認知負擔，提升操作流暢度，讓編輯流程更直覺，也讓新手使用者能更快上手。

資訊架構 Define

在視覺設計前，先盤點並統整功能規格，解決不一致處並建立資訊架構，協助團隊快速理解產品結構。透過 UI Flow 視覺化頁面路徑，確保體驗一致與流程順暢。

User Flow Ideate

以流程優化為核心，提升編輯效率與使用體驗

在釐清專案目標後，我繪製優化前後的 User Flow，標示關鍵調整點，協助團隊建立共識。例如在照片新增流程中，將原本繁瑣的操作改為可直接於編輯區上傳與替換，讓流程更直覺精簡。

同時依不同產品功能與需求持續微調流程節點，確保設計符合實際情境並提前排除潛在問題。

Wireframe

從草圖到線框：建立清晰介面架構

完成資訊架構後，我先在平板上快速分區構圖，再於 Figma 製作無色彩干擾的線框圖，聚焦在元件佈局與資訊呈現。初步草圖確認方向後，進一步繪製更精緻的版本供團隊討論，確保流程與內容無誤，再進入視覺設計階段。

UI Design

設計風格進化 Ideate

  • 分層概念
    新版採用卡片式設計，將左側照片池與編輯區等內容明確分隔，讓資訊架構更清晰，使用者能直覺理解畫面層次。
  • 簡化視覺
    透過更溫和的配色與降低對比，讓畫面聚焦於內容本身，避免過度干擾，提供更舒適的編輯體驗。
  • 清晰的點擊區域
    調整按鈕尺寸與間距，使點擊更容易辨識與操作，不僅提升可讀性，也降低誤觸的可能。
  • 一致性與層次
    重新規劃標題、選單與圖像區塊的設計，讓資訊層級更分明，整體風格保持一致，提升介面的專業感與易用性。

基本資料 Step 01

關聯性較高的內容列在一起

▍步驟式的引導：首次創建作品的遞進引導，使用者可以清楚知道一步步要怎麼編輯作品。
▍重整資訊架構：將關聯性較高的分區，讓使用者可以依序填入對應內容。
▍自動匯入：輸入的內容資料可自動匯入作品對應區塊中。

基本資料

輸入的基本資料內容可自動匯入作品中

▍輸入已知資訊，系統自動匯入

引導使用者先輸入已知的內容，系統會自動將資訊匯入，讓使用者在進入編輯畫面時，無需逐一點擊輸入框新增或更新資訊，大幅提升操作效率。

套用主題 Step 02

優化畫面引導，聚焦選擇主題並正確套用

▍簡化頁面內容，凸顯主次層級

調整按鈕配置，使畫面聚焦在「下一步：編輯內容」，移除原本的「繼續編輯」及 Header 中的「完稿並購買」，統一至編輯內容頁面後才能進行最終確認與購買，提高操作一致性與流暢度。

套用主題

主題變更確認機制，避免誤套錯誤版型

▍降低版型變更風險

首次套用或切換主題時，系統皆會跳出確認視窗，提醒使用者再次確認是否套用或變更版型，避免誤套錯誤版型並減少內容誤改，讓操作更安心。

編輯內容 Step 03

打造現代化介面：提升可讀性、視覺層次與操作直覺性

  • 資訊架構更直覺，視線動線更流暢 → 照片池統一放左側，減少視線移動。
  • 配色更現代、柔和且高級感提升 → 低飽和度色彩，減少視覺疲勞。
  • 字體與間距優化 → 提升可讀性與點擊舒適度。
  • 細節更精緻 → 圓角、陰影、視覺層次感增強。
  • 功能設計更直覺，操作更流暢 → 按鈕與工具排列更有邏輯，減少學習成本。

編輯內容

增加友善的 Persistent Toast & Tooltips

▍頁面提示框（Persistent Toast）
在需要額外輔助說明的頁面，右下角會出現提示框於畫面最上層，提醒使用者關鍵操作或注意事項，並可在閱讀後自由關閉。

▍功能按鈕引導（Tooltips）
針對操作較複雜的功能，透過 Tooltip 提供用途與操作方式的提示，並附上「知道了」按鈕，方便使用者自行關閉。

編輯內容

重整 Toolbar & Modal 元件

▍一致化視覺風格
統一文字框、QR Code、相片框等各類 Toolbar 與 Modal 元件，建立一致的視覺語言，強化介面整體性。

▍優化操作邏輯
新增雙擊文字框的進階編輯 Modal，使用者可於同一視窗調整文字、顏色、對齊與字型等設定，補足原需分步操作的限制，提升操作效率與彈性。

左側直列排列可同時顯示更多照片，並於同一畫面查看上傳進度。相較於底部欄位，側邊欄在小螢幕上更節省空間，保持畫面整潔。

Style Guide

📌視覺層次清晰，資訊更有條理

[Before]
介面元素較密集，顏色較重，資訊層次不夠明顯，且視線需要來回移動，增加認知負擔。

[After]
採用更明確的分區設計

  • 左側邊欄：照片池與管理功能統一集中，操作邏輯更直觀。
  • 主要編輯區：留白更多，內容層級更清楚，讓使用者可以專注於編輯工作。
  • 工具欄區塊分明：上方選單與左側功能區塊排列更一致，減少雜亂感。

📌色彩與對比更簡潔，符合現代設計趨勢

[Before]
採用深灰與橘色的深色背景，並透過漸層營造立體感，但強烈的色彩對比容易造成視覺疲勞，同時功能區塊之間的區隔不夠明顯。

[After]
改用更柔和、現代的配色（淺米色＋細緻灰＋低飽和橘色），整體視覺更溫暖親和，符合當代 UI 設計趨勢。背景顏色變淺，使內容與圖片更突出，避免背景喧賓奪主。對比度經過優化，畫面更柔和卻仍保持清晰可讀性。

📌圓角與留白強化層次，細節更具質感

[Before]
版面呈現早期網頁設計風格，以銳利邊角與硬性分隔為主，缺乏柔和的視覺層次。

[After]
元件與區塊改用圓角設計，介面更親和現代。適度留白提升呼吸感，減少壓迫並突顯主要內容；搭配細緻陰影與分層，清楚區隔照片編輯區與工具欄，強化視覺層次。

📌字級與排版更精緻，提升可讀性

[Before]
按鈕與文字區塊稍顯擁擠，字體大小與間距略小，導致可讀性較差。

[After]
標題更明顯，字距更適中，將關聯性較高的內容集中在同區塊。按鈕與功能名稱間距拉開，減少擁擠感，提高可讀性。

Takeaways

這次改版讓我體會到 UX 的核心在於解決痛點而非僅美化介面。我針對資訊架構與操作不順，重整照片區域、優化視覺層次與配色，並調整功能列以降低認知負擔。

透過精準提問，挖掘用戶的真實需求與痛點

與使用者訪談時，我學到關鍵不僅在詢問需求，而是透過開放式提問挖掘真實痛點。像部分用戶覺得介面「不好用」，實際原因是照片管理分散，需頻繁切換視窗。

如何建立清晰的資訊架構，讓內容有層次且易讀

在優化介面過程中，我體會到資訊架構對使用行為的影響。新版統一照片管理區域位置，減少視線移動，並透過色彩與間距強調主要功能，讓操作更順暢直覺。

反思使用者操作行為，分析背後的設計問題

除了操作不順外，我也思考使用者為何會誤操作或忽略步驟。像舊版按鈕間距過小導致誤點，問題不在用戶而是設計需優化，讓我更關注降低認知負擔與提升操作直覺性。

跨部門溝通與協作，提升效率並精準傳達設計思維

與開發與產品團隊合作時，我學會以明確邏輯表達設計需求，確保開發能準確實作。以「拖拉照片排序」為例，我運用流程圖與互動示意協助團隊快速對齊，降低溝通與修改成本。
