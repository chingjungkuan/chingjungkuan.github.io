痛點與觀察 Pain Point & Task

舊版問題

  • 文字與按鈕間距不足 → 降低可讀性，點擊易出錯。
  • 操作不直覺 → 主要按鈕不明顯，使用者需要額外摸索，學習成本高。
  • 視覺風格過時 → 缺乏現代 UI 元素，層次感不足，長時間使用容易造成視覺疲勞。
  • 資訊架構混亂 → 功能分散，使用者必須頻繁移動視線尋找，操作效率低落。
  • 照片上傳流程不順暢 → 點擊「新增照片」需跳轉頁面後再返回當前編輯畫面，且照片池以左右切換呈現，違反常見的下滑操作習慣，增加理解與操作成本。

我們發現使用者在操作過程中常遇到三大問題：難以快速找到所需資訊、學習成本高、操作流程不直覺。為了釐清根源，我們結合 Personas、Empathy Map 與 User Story Map，具體描繪出不同角色在編輯過程中的需求與行為痛點，並透過用戶訪談與實際操作觀察驗證假設，例如「照片上傳流程過於繁瑣」與「功能按鈕分散導致視線頻繁切換」。

設計優化重點 Solution

根據分析結果，我們針對核心功能進行優化：

  • 統一操作區塊位置
    將照片池固定於左側，並整合編輯工具列，減少視線與操作的跳轉
  • 優化照片新增流程
    使用者可直接於編輯區上傳與替換照片，流程更簡潔
  • 重整按鈕與工具排列邏輯
    依使用場景分類，提升操作直覺性
  • 提升視覺一致性與精緻度
    使用低飽和色系、加強層次感（圓角、陰影等），打造更現代的介面風格
  • 調整字體與元件間距
    改善閱讀與點擊體驗，提升整體易用性

這些設計有效降低了使用者的認知負擔，提升操作流暢度，讓編輯流程更直覺，也讓新手能更快上手。

同時依不同產品功能與需求持續微調流程節點，確保設計符合實際情境並提前排除潛在問題。

UI Design

設計風格進化

  • 分層概念
    新版採用卡片式設計，將左側照片池與編輯區等內容明確分隔，讓資訊架構更清晰，使用者能直覺理解畫面層次。
  • 簡化視覺
    透過更溫和的配色與降低對比，讓畫面聚焦於內容本身，避免過度干擾，提供更舒適的編輯體驗。
  • 清晰的點擊區域
    調整按鈕尺寸與間距，使點擊更容易辨識與操作，不僅提升可讀性，也降低誤觸的可能。
  • 一致性與層次
    重新規劃標題、選單與圖像區塊的設計，讓資訊層級更分明，整體風格保持一致，提升介面的專業感與易用性。

情境式操作引導

▍Coach Mark — 新功能的主動導覽

使用者進入編輯畫面時，即可看到圖文並茂的教學提示，協助快速理解新功能或關鍵操作。提示內容會依產品性質動態調整，提供最相關的引導資訊。

▍Persistent Toast — 任務過程的常駐提示

在需要額外輔助說明的頁面，右下角會出現提示框於畫面最上層，提醒使用者關鍵操作或注意事項，並可在閱讀後自由關閉。

▍Interactive Tooltips — 區塊性的即時指引

針對操作較複雜的功能，透過 Tooltip 提供用途與操作方式的提示，並附上「知道了」按鈕，方便使用者自行關閉。

▍統一現代視覺風格

統一文字框、QR Code、相片框等各類 Toolbar 與 Modal 元件，建立一致的視覺語言，強化介面整體性。

▍新增雙擊文字框的進階編輯 Modal

使用者可於同一視窗調整文字、顏色、對齊與字型等設定，補足原需分步操作的限制，提升操作效率與彈性。

照片池提高排版彈性與可擴充性

▍照片池更多操作空間
  • 調整前：照片池位於底部，空間受限且僅能瀏覽與單張刪除，易干擾編輯畫面。
  • 調整後：照片池移至左側欄，操作更集中，新增排序、篩選與多選刪除功能，管理更高效且具擴充彈性。
▍側邊欄彈性收合設計
  • 調整前：底部照片池遮擋編輯區，操作不便。
  • 調整後：改為可收合的左側欄，避免遮擋並可依需求調整空間，小螢幕操作更靈活、視線更集中。

提升行動裝置操作直覺性與流暢度

▍跨裝置響應式介面優化
  • 重新設計跨裝置介面架構，解決手機與平板上按鈕大小不一、介面擁擠的問題。
  • 改善前： 手機與平板介面擁擠、操作混亂，使用者容易誤觸且需頻繁縮放，導致整體體驗不佳。
  • 改善後： 調整按鈕大小與間距，新增平板中斷點與手勢支援，操作更為直覺流暢。
▍功能分組邏輯重整
  • 以使用者實際編輯情境為核心重新分群功能，提升手機版介面的操作邏輯。
  • 改善前：功能排列未經情境化分組，使用者在小螢幕上尋找功能較為費時，不符合行動裝置習慣。
  • 改善後：重整畫面分組，將內容操作與視圖工具邏輯化，讓編輯流程更直覺，大幅降低認知負擔。
▍觸控與操作空間優化
  • 優化操作區塊的元件尺寸與間距，並新增照片篩選等功能以提升操作順手度。
  • 改善前： 操作空間擁擠，按鈕因過小或過密而產生「難點」問題，導致觸控反應不靈敏且易出錯。
  • 改善後： 調整元件比例與熱區間距，新增照片篩選功能讓操作更精準，提升整體觸控的滑順感。

Style Guide

視覺層次清晰，資訊更有條理

Before

介面元素較密集，顏色較重，資訊層次不夠明顯，且視線需要來回移動，增加認知負擔。

After

採用更明確的分區設計

  • 左側邊欄：照片池與管理功能統一集中，操作邏輯更直觀。
  • 主要編輯區：留白更多，內容層級更清楚，讓使用者可以專注於編輯工作。
  • 工具欄區塊分明：上方選單與左側功能區塊排列更一致。

色彩與對比更簡潔，符合現代設計趨勢

Before

採用深灰與橘色的深色背景，並透過漸層營造立體感，但強烈的色彩對比容易造成視覺疲勞，同時功能區塊之間的區隔不夠明顯。

After

  • 改用更柔和、現代的配色（淺米色＋細緻灰＋低飽和橘色），整體視覺更溫暖親和，符合當代 UI 設計趨勢。
  • 背景顏色變淺，使內容與圖片更突出，避免背景喧賓奪主。
  • 畫面更柔和卻仍保持清晰可讀性。

圓角與留白強化層次，細節更具質感

Before

版面呈現早期網頁設計風格，以銳利邊角與硬性分隔為主，缺乏柔和的視覺層次。

After

  • 元件與區塊改用圓角設計，介面更親和現代。
  • 適度留白提升呼吸感，減少壓迫並突顯主要內容。
  • 搭配細緻陰影與分層，清楚區隔照片編輯區與工具欄，強化視覺層次。

字級與排版更精緻，提升可讀性

Before

按鈕與文字區塊稍顯擁擠，字體大小與間距略小，導致可讀性較差。

After

  • 標題更明顯，字距更適中，將關聯性較高的內容集中在同區塊。
  • 按鈕與功能名稱間距拉開，減少擁擠感，提高可讀性。

Takeaways

這次改版讓我體會到 UX 的核心在於解決痛點而非僅美化介面。我針對資訊架構與操作不順，重整照片區域、優化視覺層次與配色，並調整功能列以降低認知負擔。

透過精準提問，挖掘用戶的真實需求與痛點

與使用者訪談時，我學到關鍵不僅在詢問需求，而是透過開放式提問挖掘真實痛點。像部分用戶覺得介面「不好用」，實際原因是照片管理分散，需頻繁切換視窗。

如何建立清晰的資訊架構，讓內容有層次且易讀

在優化介面過程中，我體會到資訊架構對使用行為的影響。新版統一照片管理區域位置，減少視線移動，並透過色彩與間距強調主要功能，讓操作更順暢直覺。

反思使用者操作行為，分析背後的設計問題

除了操作不順外，我也思考使用者為何會誤操作或忽略步驟。像舊版按鈕間距過小導致誤點，問題不在用戶而是設計需優化，讓我更關注降低認知負擔與提升操作直覺性。

跨部門溝通與協作，提升效率並精準傳達設計思維

與開發與產品團隊合作時，我學會以明確邏輯表達設計需求，確保開發能準確實作。以「拖拉照片排序」為例，我運用流程圖與互動示意協助團隊快速對齊，降低溝通與修改成本。
