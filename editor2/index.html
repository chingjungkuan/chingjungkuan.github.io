痛點與觀察 Pain Point & Task

舊版編輯器的問題

  • 文字與按鈕間距不足 → 降低可讀性，點擊容易出錯。
  • 操作不直覺 → 主要按鈕不明顯，使用者需要額外摸索，學習成本高。
  • 視覺風格過時 → 缺乏現代 UI 元素，層次感不足，長時間使用容易造成視覺疲勞。
  • 資訊架構混亂 → 功能分散，使用者必須頻繁移動視線尋找，操作效率低落。
  • 照片上傳流程不順暢 → 點擊「新增照片」需跳轉頁面後再返回當前編輯畫面，且照片池以左右切換呈現，違反常見的下滑操作習慣，增加理解與操作成本。

我們發現使用者在操作過程中常遇到三大問題：難以快速找到所需資訊、學習成本高、操作流程不直覺。為了釐清根源，我們結合 Personas、Empathy Map 與 User Story Map，具體描繪出不同角色在編輯過程中的需求與行為痛點，並透過用戶訪談與實際操作觀察驗證假設，例如「照片上傳流程過於繁瑣」與「功能按鈕分散導致視線頻繁切換」。

設計優化重點 Solution

為了解決上述問題，我重新規劃了操作流程與介面結構，讓整體體驗更直覺清晰：

  • 統一操作區塊位置：將照片池固定於左側，並整合編輯工具列，減少視線與操作的跳轉
  • 優化照片新增流程：使用者可直接於編輯區上傳與替換照片，流程更簡潔
  • 重整按鈕與工具排列邏輯：依使用場景分類，提升操作直覺性
  • 提升視覺一致性與精緻度：使用低飽和色系、加強層次感（圓角、陰影等），打造更現代的介面風格
  • 調整字體與元件間距：改善閱讀與點擊體驗，提升整體易用性

這些設計有效降低了使用者的認知負擔，提升操作流暢度，讓編輯流程更直覺，也讓新手使用者能更快上手。

資訊架構 Define

在視覺設計前，先盤點並統整功能規格，解決不一致處並建立資訊架構，協助團隊快速理解產品結構。透過 UI Flow 視覺化頁面路徑，確保體驗一致與流程順暢。

User Flow Ideate

以流程優化為核心，提升編輯效率與使用體驗

在釐清專案目標後，我繪製優化前後的 User Flow，標示關鍵調整點，協助團隊建立共識。例如在照片新增流程中，將原本繁瑣的操作改為可直接於編輯區上傳與替換，讓流程更直覺精簡。

同時依不同產品功能與需求持續微調流程節點，確保設計符合實際情境並提前排除潛在問題。

Wireframe

從草圖到線框：建立清晰介面架構

完成資訊架構後，我先在平板上快速分區構圖，再於 Figma 製作無色彩干擾的線框圖，聚焦在元件佈局與資訊呈現。初步草圖確認方向後，進一步繪製更精緻的版本供團隊討論，確保流程與內容無誤，再進入視覺設計階段。

UI Design

設計風格進化 Ideate

  • 分層概念（Elevation & Layering）新版使用了卡片式設計，將內容明確區分（如左側照片池、編輯區），這與 Material Design 中的「分層」概念相符，讓使用者直覺理解資訊架構。
  • 簡化視覺（Minimal & Functional Aesthetic）新版配色更溫和、對比減少，視覺更舒服，這符合 Material Design 提倡的 「不過度裝飾、強調內容」 原則。
  • 清晰的點擊區域（Touchable & Readable）按鈕與點擊區域間距變大，可讀性提升，不容易誤點，這符合 Material Design 的「大面積觸控區域」概念，方便使用者操作。
  • 一致性與層次（Hierarchy & Consistency）標題、選單、圖像區塊明確，資訊層次感提升，這與 Material Design 強調的「一致性」原則相符，讓 UI 更直覺易懂。

Style Guide

📌視覺層次更清晰，資訊組織更有條理

1.0： 介面元素較密集，顏色較重，資訊層次不夠明顯，且視線需要來回移動，增加認知負擔。

2.0： 採用更明確的分區設計，如： 左側邊欄：照片池與管理功能統一集中，操作邏輯更直觀。 主要編輯區：留白更多，內容層級更清楚，讓使用者可以專注於編輯工作。 工具欄區塊分明：上方選單與左側功能區塊排列更一致，減少雜亂感。

📌使用圓角、留白與陰影，提升設計細節與質感

1.0： 版面偏向早期網頁設計風格，較多銳利邊角與硬性分隔，缺乏柔和的視覺層次。

2.0： 使用圓角設計，如按鈕、卡片、選單，讓介面更友善與現代化。 適當留白，提升整體的呼吸感，減少壓迫感，讓用戶更容易專注在主要內容上。 細膩的陰影與分層，例如照片編輯區與工具欄之間的視覺區隔更清晰，避免資訊混亂。

📌 色彩與對比更簡潔，符合現代 UI 設計趨勢

1.0： 深色背景（深灰 + 橘色）加上就有的設計風格是透過間曾創造立體感，但對比強烈，容易造成視覺疲勞，且功能區塊之間的區隔不夠明顯。

2.0： 採用更柔和、現代的配色（淺米色 + 細緻的灰色 + 低飽和橘色），整體視覺更溫暖、親和，符合當代 UI 設計趨勢> 背景顏色變淺，讓內容與圖片更突出，而非背景喧賓奪主。 對比度優化，讓視覺更柔和但仍保持清晰可讀性。

📌 字體設計與排版更精緻，提升可讀性

1.0： 按鈕與文字區塊稍顯擁擠，字體大小與間距略小，導致可讀性較差。

2.0： 標題更明顯，字距更適中，將關聯性較高的內容集中在同區塊，使用更細緻的字體與留白。 按鈕與功能名稱間距拉開，減少擁擠感，提高可讀性。

Takeaways
這次改版讓我更深入理解產品設計的核心不只是畫面美觀，而是針對使用者痛點提出具體解方。針對舊版介面資訊架構混亂、操作不順的問題，我重新規劃照片管理區域、優化視覺層次與配色，並調整功能列位置，讓操作路徑更直覺，明顯降低使用者的認知負擔與學習門檻。

如何透過精準提問，挖掘用戶的真實需求與痛點

與使用者訪談時，我學習到不僅僅是詢問他們的需求，而是透過開放式問題引導，挖掘他們操作時的不便與真正的困擾。例如，部分用戶認為介面「不好用」，但透過深入詢問與觀察，發現問題其實是「照片管理方式過於分散，導致使用者需要頻繁切換視窗」。

如何建立清晰的資訊架構，讓內容有層次且易讀

在優化介面時，我深刻體會到資訊架構對使用者行為的影響。新版設計中，我統一了照片管理區域的位置，減少使用者視線來回移動的次數，並透過顏色與間距的調整，讓主要功能更顯眼，確保操作流程更順暢。

反思使用者操作行為，分析背後的設計問題

除了直觀的操作不順外，我也開始思考使用者為什麼會犯錯、為什麼某個步驟會被忽略。例如，舊版介面按鈕間距擁擠，導致誤點率高，而這並非用戶本身的錯誤，而是 UI 設計上的改進空間。這讓我更關注「減少認知負擔」與「提升操作直覺性」。

跨部門溝通與協作，提升效率並精準傳達設計思維

在與開發與產品團隊的合作中，我學會了如何用明確的邏輯表達設計需求，確保開發端能準確理解並實作。例如，當討論「拖拉照片排序」的功能時，我透過具體的流程圖與交互示意，讓團隊能快速對齊共識，減少來回調整的成本。
